# 카카오 키 받는 법 (천천히 따라하면 5분)

> **키가 없어도 앱은 돌아간다.** 지금은 "간이 지도 + 서울 샘플 장소 검색"으로 동작하고,
> 아래 카카오 키를 넣으면 **진짜 카카오 지도 + 전국 장소 검색**으로 자동으로 바뀐다.
> Supabase(같이/투표 모드용)는 아직 안 쓰니 지금은 신경 쓰지 않아도 된다.

---

## 카카오맵 JavaScript 키

### 1. 카카오 디벨로퍼스 가입

1. https://developers.kakao.com 접속
2. 오른쪽 위 **로그인** → 평소 쓰는 카카오 계정으로 로그인
3. 처음이면 개발자 약관 동의 한 번 하면 끝

### 2. 앱 만들기

1. 상단 메뉴 **내 애플리케이션** 클릭
2. **애플리케이션 추가하기** 버튼
3. 입력:
   - **앱 이름**: 아무거나 (예: `동선캘린더`)
   - **사업자명**: 아무거나 (예: 본인 이름)
4. **저장**

### 3. JavaScript 키 복사

1. 방금 만든 앱을 클릭
2. 왼쪽 메뉴 **앱 키**
3. 거기 있는 **JavaScript 키**를 복사한다. (REST API 키 아님, **JavaScript 키**)

### 4. 사용할 도메인 등록 (이게 빠지면 지도가 안 뜬다)

1. 왼쪽 메뉴 **플랫폼 키 → JavaScript 키 → JS SDK 도메인** (또는 **플랫폼 → Web**)
2. 아래 두 줄을 각각 추가:
   ```
   http://localhost:3000
   http://127.0.0.1:3000
   ```
3. 저장

> ⚠️ **다른 포트로 실행하면** 그 포트도 등록해야 지도가 뜬다. 예: `PORT=3100`으로 실행 중이면
> `http://localhost:3100`, `http://127.0.0.1:3100`도 추가. (자동차·대중교통·도보는 서버에서
> 부르므로 포트와 무관하게 동작한다.)

### 5. 키를 앱에 넣기

1. 프로젝트 폴더에서 `.env.local.example` 파일을 복사해 `.env.local` 로 이름을 바꾼다.
   ```bash
   cp .env.local.example .env.local
   ```
2. `.env.local` 을 열어 복사한 JavaScript 키를 붙여넣는다:
   ```
   NEXT_PUBLIC_KAKAO_JS_KEY=여기에_복사한_JavaScript_키
   ```
3. 저장하고, 개발 서버를 껐다 켠다 (`Ctrl+C` 후 `npm run dev`).
   - 환경변수는 서버 재시작해야 반영된다.

이제 앱을 새로고침하면 간이 지도가 **진짜 카카오 지도**로 바뀌고,
검색창에서 전국 어디든(예: "강남역 스타벅스") 검색된다.

## ODsay 키 — 대중교통(지하철/버스) 길찾기용

카카오 무료 API는 **자동차·도보**만 되고 **대중교통 경로(몇 호선, 몇 번 버스, 환승)**는 못 준다.
그래서 대중교통은 ODsay라는 무료 API를 쓴다. (무료 플랜 하루 1,000회)

### 1. 가입

1. https://lab.odsay.com 접속
2. **회원가입** → 이메일로 가입하고 로그인

### 2. 인증키 발급 (서비스 등록)

1. 로그인 후 **마이페이지** → **API 인증키 발급** (또는 "웹 서비스 등록")
2. 새 서비스(애플리케이션) 등록:
   - **서비스명**: 아무거나 (예: `동선캘린더`)
   - **서비스 URL / 도메인**: `http://localhost:3000` (배포하면 그때 실제 주소 추가)
3. 등록하면 **인증키(API Key)** 가 나온다. 이걸 복사.
   - 키에 특수문자가 있을 수 있는데, 앱이 알아서 처리하니 **그대로** 복사하면 된다.

### 3. 키를 앱에 넣기

`.env.local` 파일을 열어 한 줄 추가한다(카카오 키 아래에):

```
ODSAY_KEY=여기에_복사한_ODsay_인증키
```

저장하고 개발 서버를 껐다 켠다 (`Ctrl+C` 후 `npm run dev`).
이제 앱에서 **🚇 대중교통** 버튼을 누르면 구간마다 "몇 호선 / 몇 번 버스 / 몇 분"이 나온다.

> ODsay 키도 **비밀 키**다(카카오 REST 키처럼). 그래서 `NEXT_PUBLIC_` 없이 넣어 서버에서만 쓴다.
> 막히면: 대부분 서비스 URL에 `http://localhost:3000`을 안 넣었거나 키 오타다.

## TMap 키 — 도보(보행자) 경로용

카카오는 **자동차**만, ODsay는 **대중교통**만 준다. **실제 보행 경로(인도 기준)**는 TMap의
보행자 길찾기 API를 쓴다. (SK Open API, 무료 키)

### 1. 가입 / 프로젝트 만들기

1. https://openapi.sk.com 접속 → 회원가입 후 로그인
2. 상단 **프로젝트(또는 My Project)** → **프로젝트 생성**
   - 프로젝트 이름: 아무거나 (예: `동선캘린더`)

### 2. TMAP API 추가하고 키 받기

1. 만든 프로젝트에서 **API 추가/구독** → **TMAP** 카테고리에서 **보행자 경로안내(Pedestrian)** 추가
2. 프로젝트 화면의 **App Key**(앱키)를 복사한다.

### 3. 키를 앱에 넣기

`.env.local` 에 한 줄 추가하고 서버 재시작:

```
TMAP_APP_KEY=여기에_복사한_TMap_AppKey
```

이제 **🚶 도보**를 고른 구간은 직선이 아니라 **실제 보행 경로**를 따라 그려진다.

> TMap appKey도 비밀로 취급해 서버에서만 쓴다(`NEXT_PUBLIC_` 없음).
> 보행자 경로는 한 구간 거리 제한이 있어(대략 수십 km), 너무 먼 구간은 직선 추정으로 자동 폴백한다.

## (나중에) Supabase — 계정·동기화·사진·공유 단계에서 필요

지금은 브라우저(localStorage)에만 저장하므로 **아직 필요 없다.** 다음 단계에서 쓰게 된다:
여러 기기 동기화 · 백그라운드 알림 · **다이어리 사진 저장** · "같이 정하기(공유·투표)".
그때 https://supabase.com 에서 무료 프로젝트를 만들고 `Project URL` 과 `anon public` 키를 쓴다.
(그 단계에 들어가면 이 문서에 상세 절차를 추가한다.)

## 자주 막히는 곳

- **지도가 회색으로만 보임** → 4번 도메인 등록을 안 했거나 오타. `http://localhost:3000` 정확히.
- **키 넣었는데 그대로임** → 서버를 껐다 켰는지 확인. `.env.local` 파일 이름이 정확한지 확인(`.example` 빼기).
- **REST 키를 넣음** → 이 앱은 **JavaScript 키**를 쓴다. 앱 키 화면에서 JavaScript 키를 다시 복사.

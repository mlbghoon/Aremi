# MVP — 실제 만든 것 (동선 캘린더)

> 이 문서는 **지금까지 실제로 구현한 MVP**를 정리한다. 큰 그림은
> [`route-calendar-app-plan.md`](route-calendar-app-plan.md), 실행/키/구조는 루트 [`../README.md`](../README.md).

## 방향이 어떻게 잡혔나

초기 MVP 아이디어는 "혼자 내 동선 짜기 + 같이 투표로 코스 정하기(웹 공유)"였다.
만들면서 **개인용 일정 관리로 방향이 굳어졌다.** 이유는 명확하다 — 매일 여는 습관을 만드는 건
'투표'가 아니라 '내 일정과 동선'이기 때문. 그래서 지금 MVP는 이렇게 정리됐다.

> **미리 일정만 넣어두면, 지도에서 다시 검색할 필요 없이 그날의 동선과 출발 시각이 바로 나온다.
> 지나간 날은 다이어리로 돌아본다.**

'같이 정하기(투표·공유)'는 버린 게 아니라 **로드맵 뒤로** 옮겼다(클라우드 동기화 이후).

## 만든 것

### 1. 캘린더 (월 / 주 / 일)

- 날짜를 골라 그날 일정을 관리한다. 월은 칸에 일정 칩, 주·일은 시간대 격자에 블록.
- 일정 = **이름 + 장소 + 시간(시작~종료 또는 '아무때나') + 메모 + 반복 + 출발 알림**.
- 반복: 안 함 / 매일 / 평일 / 매주. 반복 일정은 여러 날에 자동으로 나타난다.

### 2. 동선 (핵심)

- "시간 고정" 약속은 그 순서를 지키고, "아무때나"는 그 사이를 가장 효율적인 순서로 자동 배치.
- 구간마다 **자동차 / 대중교통 / 도보**를 고르면 실제 경로를 지도에 색으로 그린다.
  - 자동차: 카카오모빌리티 도로 경로
  - 대중교통: ODsay 지하철/버스 + 실제 노선 좌표
  - 도보: TMap 보행자 경로
- **미리 넣은 일정이 곧 경로의 출발지·도착지** — 지도에서 다시 검색할 필요가 없다.

### 3. 스마트 시간표 + 출발 알림

- 실제 이동시간으로 각 장소의 **도착/출발 시각**을 계산하고, 고정 약속에 늦으면 경고한다.
- 직전 일정에서 여기까지 걸리는 시간을 반영한 **출발 알림**("지금 출발"). 앱이 열려 있을 때 동작.

### 4. 다이어리(기록)

- 동선 화면에서 `🗺 동선` ↔ `📖 기록` 토글.
- 기록은 "이날 뭐 했지?"를 **항목 + 메모 + 사진 + 지도**로 보여준다.
- 사진은 지금 **기기(IndexedDB)에만 임시 저장** — 여러 기기 동기화·영구 클라우드 백업은 백엔드(Supabase) 단계.

### 5. 그 외

- 폰 프레임 다크 테마 모바일 앱 UI, 바텀시트 편집.
- 브라우저(localStorage) 자동 저장 — 앞으로의 계획이 새로고침해도 남는다.

## 일부러 뺀 것 / 다음

- **계정 + 클라우드 동기화(Supabase)** — 여러 기기, 백그라운드 알림, **사진 클라우드 저장**의 기반.
  (사진 첨부 UI 자체는 이미 있음 — 지금은 기기 임시 저장이라 영구·다기기 백업이 이 단계의 몫.)
- **같이 정하기(공유·투표)** — 원래 기획의 '같이' 모드. 동기화 이후.
- **실시간 위치 공유(약속 모드)** — 큰 그림 문서의 4-4. 훨씬 뒤.
- 첫 일정의 출발 알림(출발 위치를 모름 → 이전 일정이 있어야 계산).

## 성공 판단 (여전히 유효)

- 내가 하루 동선을 짤 때 이걸 켜는가.
- 다음 약속·일정을 잡을 때 실제로 이걸 쓰는가.
- 지나간 날을 여기서 돌아보는가.
